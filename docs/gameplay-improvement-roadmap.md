## 1. コード接地レビュー（調査の想定 vs 実態）

調査レポートが「未確認」とした点を、実装を読んで確定・補正した。

| 論点 | レポート想定 | コード実態 | コスト補正 |
|---|---|---|---|
| **発見の永続化** | （言及なし） | `foundControllerProvider` は **autoDispose のインメモリ**。永続化されるのは「シーンのクリア」だけで、**「何を見つけたか」はシーンを出ると消える** | ★コレクション系の前提が欠落 |
| C1 配置シャッフル | 低（アセット0） | 配置は **JSON にベタ書き**。「入場時にアイコン↔位置を入替」で実現。点滅位相は slot 番号基準で位置入替と無干渉 | **XS**。最年少配慮で**再訪時のみ**推奨 |
| D6 影絵→カラー | 低 | 図鑑バーは既に「未発見=グレー / 発見=カラー」。真の不足は**横断的な図鑑画面が無い**こと | 永続化前提で **S** |
| D1 シール帳 | 中 | 発見の永続化が無いため**保存層＋新画面**が必要 | **M〜L** |
| B6 ラスト / A5 連鎖 / B1 セレモニー | 低 | **`FoundBurst` と `_ClearOverlay` が既存**。分岐・カウント・confetti を足すだけ | **XS〜S**（即効） |
| A2 どかし露出 | 低 | なぞりは「うごかす/なぞる」トグル＋InteractiveViewer と競合整理済み。発見ドラッグ追加で再複雑化 | **M に格上げ** |
| F3/F7/B5 解放拡張 | 低 | `TrailStyle.unlockRequirement` / `syncTrailUnlocks` の解放フックが既存 | **XS** |
| A1 めくり露出 | 低 | 宝に「カバー状態」＋シーンデータ＋ヒット判定の状態追加 | **S〜M** |

**最重要**: 調査は「収集が本丸」と正しく言い当てたが、その実装前提＝**“発見した宝の永続化”が現状ゼロ**である点は拾えていない。これを小さく作るのが、コレクション系（D1/D6/D7/D8・C8・F3・B4）を**一括解錠する keystone**。

## 3. フェーズ別ロードマップ

研究の Top3（C1/D1/A1）を、コード実態で順序組み替え。juice は無料で即効・D1 は keystone を挟む・C1 は再訪限定が安全。

> **実装状況（2026-06-21）**: Phase 0・Phase 1・Phase 2（keystone + D6 + D8）まで実装済み（PR #49〜#52）。
> Phase 2 の残り（D1 シール帳・C8・B4・F3）と Phase 3・4 は今後。

## 4. まず着手する3点（推奨）

1. **Phase 0 の B6＋A5＋B1**（即効・ほぼ無料で手応え）
2. **Phase 2 の keystone（発見の永続化）**（コレクション群を一括解錠する投資）
3. **C1 再訪シャッフル**（アセット0のリプレイ性）

→「即効の手応え（juice）」「最大の伸びしろへの布石（収集）」「無料のリプレイ性」を最小コストで同時に押さえる。

## 5. ガードレール（各 PR のチェック項目）

調査の "やってはいけないリスト" を、不変原則の裏返し＝**実装時チェック**に落とす。

- 連鎖が途切れても**減点・崩壊音を出さない**（静かにリセットのみ）
- お題・収集に**制限時間 / 期間限定 / ストリークを付けない**
- 収集を**課金・トークン・ガチャに接続しない**（ご褒美＝宝の絵）
- コンプ率の**他者比較をしない**（「自分の図鑑」の所有感のみ）
- お題は**必ず絵＋音**（読字非依存）
- 演出は**山場だけ豪華・常時は控えめ**（calm 哲学・過剰刺激回避）
