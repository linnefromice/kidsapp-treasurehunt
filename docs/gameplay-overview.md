# ゲーム性インベントリ（シーク＆ファインド）

現状のゲーム機能を **ゲーム性中心** で整理したもの。改善アイデア収集の土台として使う。

> 対象時点: `main`（2026-06-21 時点 / PR #43〜#46 反映済み）。
> 設計の正典は `docs/superpowers/specs/2026-06-17-treasure-hunt-skeleton-design.md`、
> 規制・UX 基準は `README.md`、AI 運用は `CLAUDE.md` を参照。

---

## 1. コアループ

1 枚のシーンに隠れた **宝をタップ／なぞって見つける** → 図鑑バーに収納 →
**全部見つけるとシーンクリア** → 宝の地図で **次のシーンが解放** → 13 ワールドを順に踏破。

- **失敗を罰しない**設計が一貫: 空振りは控えめな「ミスバブル」のみ（×・減点・不快音なし）、
  **時間制限なし**、**無制限ヒント**、スター／スコア／競争／リーダーボードは **意図的に無し**。
- 発見の瞬間に多感覚フィードバックを集中: 拡大グロー＋キラッ（FoundBurst）＋効果音＋触覚。

## 2. 探す操作（モード共通＋差分）

- **タップで発見**（全モード共通）。Easy は押した瞬間（onTapDown）、Normal/Hard はタップ確定時（onTapUp）。
- **なぞって発見（ペン）**: 指先に色付きキラキラ（トレイル）が追従。
- **ピンチ拡大**: Normal/Hard の「うごかす」モードで可。

## 3. 難易度（3 段階・選択は永続化）

| | Easy（やさしい） | Normal（ふつう） | Hard（むずかしい） |
|---|---|---|---|
| 探索エリア | 画面ぴったり（パン不要） | 画面より広い（パン必須） | 同左 |
| おとり量 | dummies のみ（12） | dummies + hardDummies（20） | 同左 |
| 宝の状態 | 静止 | 静止 | **宝・おとりが点滅（消える/現れる）** |
| ドラッグ操作 | なぞり発見＋キラキラ | **「うごかす／なぞる」トグル**で切替 | 同左 |
| 宝の数 | 8（全モード共通・不変） | 8 | 8 |

- **Hard の点滅**: 周期 3 秒、1 回あたり約 0.9 秒消える。**消えている宝はタップ不可**
  （消失中のタップは無反応＝罰しない）。位相をずらして全部同時には消えない。
  **残り 1 つになったら点滅停止**（必ず押せる時間を確保）。

## 4. 収集・図鑑

- 画面下の **図鑑バー** は宝を **アイコン単位でグルーピング**。同じアイコンが複数あると
  「0/2」のような **カウントアップ表示**。
- 見つけると該当スロットが点灯（グレー→点灯）。

## 5. 進行・ワールド地図

- 羊皮紙風の **宝の地図** に 13 ワールドが曲線ルートで配置。ノードは
  **未解放（ロック）／現在地／クリア済み** の 3 状態。
- 現在地ノードへ **足跡マーチング演出** でそっと誘導（急かさない）。
- 進行は **モードごとに独立した一本道**（Easy のクリアと Hard のクリアは別々）。
- **フリーモード**: 全シーン解放で自由に遊べる（実スロットと進捗独立）。

## 6. ヒント・誘導（読字に依存しない）

- **アイドルヒント**: 8 秒無操作で未発見の宝が 1 つ淡く光る。操作すると消える。
- 絵＋音＋アニメで誘導（テキスト非依存）。ja/en 切替対応。

## 7. カスタム＝唯一の“報酬／やり込み”軸

- **なぞりトレイルのスタイル**: たんしょく（6 色から 1 色）／にじ（3 色選択・循環）／
  にじ（フル・色相が流れる）。
- **解放条件**: にじ3色＝**Easy 全クリア**、にじフル＝**Hard 全クリア**（一度解放したら永続）。
  ← 現状ほぼ唯一の「クリアの外的報酬」。
- **アバター絵文字**（66 種から選択・後から変更可）。スロットの識別と愛着づけ。

## 8. セーブ／枠組み

- 固定 3 スロット＋フリーモード枠。**進捗はスロット独立**、言語・トレイル設定は全スロット共通。
- リセットは **保護者ゲート（算数）** 経由。アバター変更は非破壊なのでゲートなし。

## ゲーム性の俯瞰と伸びしろ候補

現状は **「発見の気持ちよさ × 探索の難度設計（量・広さ・点滅）」が core で、外的報酬は意図的に最小**
（トレイル解放とアバターのみ、スター/競争は排除）。これは調査レポート由来の
「0–5 は無報酬で成立／失敗を罰しない」を厳格に守った設計。

- **強い軸**: 発見フィードバックの濃さ、難度の段階表現（量 → 広さ → 点滅）、no-fail。
- **薄い軸（＝伸びしろ候補）**:
  - **(a) 達成感・継続動機**: 地図踏破以外の節目演出が少ない。
  - **(b) 探索のバリエーション**: 仕掛け・お題・連鎖などのメカニクスは無く、純粋な発見のみ。
  - **(c) リプレイ性**: 同一シーンを再訪する理由が薄い。
  - **(d) collection の見せ方**: 図鑑が「進捗バー」止まりで、“集めた図鑑”としての魅力は未活用。

> 改善アイデアを出す際の制約条件（不変の設計原則）: 失敗を罰しない／時間制限を入れない／
> スター・競争・リーダーボードを入れない／行動広告 SDK・データ収集ゼロ／保護者ゲート維持。
