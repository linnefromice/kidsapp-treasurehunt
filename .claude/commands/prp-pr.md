# プルリクエスト作成

> WirasmによるPRPs-agentic-engから適用。PRPワークフローシリーズの一部です。

**入力**: `$ARGUMENTS` — オプション。ベースブランチ名やフラグ（例: `--draft`）を含む場合があります。

---

## フェーズ 1 — バリデーション

前提条件をチェック:
- ベースブランチ上でないこと
- クリーンなワーキングディレクトリ
- origin/baseより先のコミットがあること
- 既存のPRがないこと

## フェーズ 2 — ディスカバリー

PRテンプレートの検索（`.github/PULL_REQUEST_TEMPLATE.md` など）、コミット分析、ファイル分析、PRPアーティファクトの確認。

## フェーズ 3 — プッシュ

```bash
git push -u origin HEAD
```

乖離によりプッシュが失敗した場合はリベースを実行。

## フェーズ 4 — 作成

テンプレートが見つかった場合はそのセクションを埋め、見つからない場合はデフォルトフォーマット（サマリー、変更、ファイル変更、テスト、関連issue）を使用。

```bash
gh pr create \
  --title "<PRタイトル>" \
  --base <base-branch> \
  --body "<PR本文>"
```

## フェーズ 5 — 検証

PRの状態とCIチェックを確認。

## フェーズ 6 — 出力

ユーザーに報告: PR番号、URL、ブランチ、変更数、CIチェック状態、次のステップ。

---

## エッジケース

- **`gh` CLIがない**: 停止: 「GitHub CLI (`gh`) が必要です。インストール: <https://cli.github.com/>」
- **未認証**: 停止: 「まず `gh auth login` を実行してください。」
- **フォースプッシュが必要**: `git push --force-with-lease` を使用（`--force` は使用しない）。
- **複数のPRテンプレート**: リストを表示してユーザーに選択を依頼。
- **大きなPR（20ファイル超）**: PRサイズについて警告。論理的に分割可能であれば分割を提案。
