# PRP Implement

計画ファイルを継続的な検証付きでステップバイステップに実行します。すべての変更は即座に検証されます — 壊れた状態を蓄積しないでください。

**コア哲学**: 検証ループが早期にミスをキャッチします。すべての変更後にチェックを実行してください。問題は即座に修正してください。

**黄金律**: 検証が失敗した場合、次に進む前に修正してください。壊れた状態を蓄積しないでください。

---

## フェーズ 0 — 検出

### パッケージマネージャーの検出

| ファイルの存在 | パッケージマネージャー | ランナー |
|-------------|-------------------|---------|
| `bun.lockb` | bun | `bun run` |
| `pnpm-lock.yaml` | pnpm | `pnpm run` |
| `yarn.lock` | yarn | `yarn` |
| `package-lock.json` | npm | `npm run` |
| `pyproject.toml` または `requirements.txt` | uv / pip | `uv run` または `python -m` |
| `Cargo.toml` | cargo | `cargo` |
| `go.mod` | go | `go` |

---

## フェーズ 1 — 読み込み

計画ファイルを読み込み、以下のセクションを抽出:
- **サマリー** — 何を構築するか
- **ミラーするパターン** — 従うべきコード規約
- **変更するファイル** — 作成または変更するもの
- **ステップバイステップタスク** — 実装シーケンス
- **検証コマンド** — 正確性の確認方法
- **受け入れ基準** — 完了の定義

---

## フェーズ 2 — 準備

### Gitの状態確認とブランチの決定

| 現在の状態 | アクション |
|-----------|----------|
| フィーチャーブランチ上 | 現在のブランチを使用 |
| main上、クリーンなワーキングツリー | フィーチャーブランチを作成: `git checkout -b feat/{plan-name}` |
| main上、ダーティなワーキングツリー | **停止** — stashまたはコミットを先に行うようユーザーに依頼 |
| この機能用のgitワークツリー内 | ワークツリーを使用 |

---

## フェーズ 3 — 実行

計画の各タスクを順次処理します。

### タスクごとのループ

各タスクについて:

1. **MIRRORリファレンスを読む** — タスクのMIRRORフィールドで参照されているパターンファイルを開く
2. **実装** — パターンに正確に従ってコードを書く。GOTCHA警告を適用。指定されたIMPORTSを使用
3. **即座に検証** — ファイル変更のたびに型チェックを実行。失敗した場合は次のファイルに進む前にエラーを修正
4. **進捗を追跡** — ログ: `[done] Task N: [task name] — complete`

---

## フェーズ 4 — 検証

計画のすべての検証レベルを実行。各レベルで問題を修正してから次に進みます。

- **レベル 1**: 静的解析（型チェック、lint）
- **レベル 2**: ユニットテスト
- **レベル 3**: ビルドチェック
- **レベル 4**: 統合テスト（該当する場合）
- **レベル 5**: エッジケーステスト

---

## フェーズ 5 — レポート

`.claude/PRPs/reports/{plan-name}-report.md` に実装レポートを作成。
該当する場合はPRDのフェーズステータスを更新。
計画を `completed/` にアーカイブ。

---

## フェーズ 6 — 出力

ユーザーに報告:
- 計画ファイル → アーカイブ先
- ブランチ名
- 検証サマリー（型チェック、lint、テスト、ビルドの各ステータス）
- 変更ファイル数
- 計画からの逸脱
- アーティファクトのパス

> 次のステップ: `/prp-pr` でプルリクエストを作成するか、`/code-review` で先に変更をレビューしてください。

---

## 成功基準

- **TASKS_COMPLETE**: 計画のすべてのタスクが実行済み
- **TYPES_PASS**: 型エラーゼロ
- **LINT_PASS**: lintエラーゼロ
- **TESTS_PASS**: すべてのテストがグリーン、新しいテストが書かれた
- **BUILD_PASS**: ビルド成功
- **REPORT_CREATED**: 実装レポートが保存済み
- **PLAN_ARCHIVED**: 計画が `completed/` に移動済み
