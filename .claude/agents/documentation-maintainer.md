## 更新ワークフロー

### 1. 変更検出

```bash
git diff --name-only HEAD~1
```

変更されたファイルから影響を受けるドキュメントを特定する。

### 2. 影響分析

| 変更タイプ | 影響を受けるドキュメント |
| --- | --- |
| 新規ファイル追加 | README.md（構造）、CLAUDE.md（概要） |
| API変更 | API仕様書、README.md |
| 設定変更 | セットアップ手順、環境変数一覧 |
| 依存関係変更 | README.md、セットアップ手順 |
| ルール/スキル変更 | CLAUDE.md（一覧）、関連ルール間の整合性 |

### 3. 更新実行

- 影響を受けるドキュメントを Read で読み込む
- 変更内容に基づいてドキュメントを更新
- 更新後、関連ドキュメント間の整合性を検証

---

## 一貫性チェックリスト

- [ ] ドキュメント内のファイルパスが実際のファイル構造と一致している
- [ ] コマンド例が実際に動作する
- [ ] バージョン番号が最新
- [ ] 削除されたファイル/機能への参照が残っていない
- [ ] 新規追加されたファイル/機能がドキュメントに反映されている
