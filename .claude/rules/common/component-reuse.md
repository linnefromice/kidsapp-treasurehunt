# コンポーネント・モジュール共通化ルール

UI要素やロジックを作成・編集する際の必須ルール。

---

## 3つ以上ルール

**同じ形・フォーマットの要素が3つ以上あるなら、必ず共通化すること。**

コンポーネントやモジュールを作成・編集する前に確認:

1. 同じ構造のUI/ロジックが他にもあるか？
2. パラメータだけ違う同じパターンが3つ以上あるか？
3. あるなら → 共通モジュールを作成

| 特定モジュールのみ | モジュール内のローカルディレクトリ |

## 共通化パターン

### データ駆動パターン

```
// BAD: 個別の関数をハードコード
function processA(data) { /* ... */ }
function processB(data) { /* ... */ }
function processC(data) { /* ... */ }

// GOOD: データ駆動の汎用関数
function processItem(data, config: ProcessConfig) {
    // config定義に基づいて処理
}
```

### 定数集約パターン

```
// BAD: マジックナンバーの散在
if (value > 100) { /* ... */ }
if (count >= 3) { /* ... */ }

// GOOD: 定数定義を集約
export const MAX_VALUE = 100;
export const MIN_COUNT = 3;
```
