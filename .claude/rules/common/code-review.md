# コードレビュー基準

## 目的

コードレビューは、コードがマージされる前に品質、セキュリティ、保守性を確保します。このルールは、コードレビューをいつ、どのように実施するかを定義します。

## レビューのタイミング

**必須レビュートリガー:**

- コードを書いたり修正した後
- 共有ブランチへのコミット前
- セキュリティに関わるコードが変更された場合（認証、決済、ユーザーデータ）
- アーキテクチャの変更が行われた場合
- プルリクエストのマージ前

**レビュー前の要件:**

レビューを依頼する前に、以下を確認してください：

- すべての自動チェック（CI/CD）がパスしていること
- マージコンフリクトが解決されていること
- ブランチがターゲットブランチの最新状態であること

## レビューチェックリスト

コードを完了とマークする前に：

- [ ] コードが読みやすく、適切な命名がされている
- [ ] 関数が焦点を絞っている（50行未満）
- [ ] ファイルにまとまりがある（800行未満）
- [ ] 深いネストがない（4レベル超）
- [ ] エラーが明示的にハンドリングされている
- [ ] ハードコードされたシークレットや認証情報がない
- [ ] console.log やデバッグ文がない
- [ ] 新機能にテストが存在する
- [ ] テストカバレッジが最低80%を満たしている

## セキュリティレビュートリガー

**以下の場合は作業を停止し、security-reviewer エージェントを使用してください：**

- 認証または認可コード
- ユーザー入力の処理
- データベースクエリ
- ファイルシステム操作
- 外部 API 呼び出し
- 暗号化操作
- 決済または金融コード

## レビュー重要度レベル

| レベル | 意味 | アクション |
|--------|------|-----------|
| CRITICAL | セキュリティ脆弱性またはデータ損失リスク | **ブロック** - マージ前に必ず修正 |
| HIGH | バグまたは重大な品質問題 | **警告** - マージ前に修正すべき |
| MEDIUM | 保守性の懸念 | **情報** - 修正を検討 |
| LOW | スタイルまたは軽微な提案 | **注記** - 任意 |

## エージェントの使用

コードレビューには以下のエージェントを使用します：

| エージェント | 目的 |
|-------------|------|
| **code-reviewer** | 一般的なコード品質、パターン、ベストプラクティス |
| **security-reviewer** | セキュリティ脆弱性、OWASP Top 10 |
| **typescript-reviewer** | TypeScript/JavaScript 固有の問題 |
| **python-reviewer** | Python 固有の問題 |
| **go-reviewer** | Go 固有の問題 |
| **rust-reviewer** | Rust 固有の問題 |

## レビューワークフロー

```
1. git diff で変更内容を把握する
2. まずセキュリティチェックリストを確認する
3. コード品質チェックリストをレビューする
4. 関連するテストを実行する
5. カバレッジ >= 80% を確認する
6. 適切なエージェントを使用して詳細レビューを行う
```

## よくある検出すべき問題

### セキュリティ

- ハードコードされた認証情報（API キー、パスワード、トークン）
- SQL インジェクション（クエリ内の文字列結合）
- XSS 脆弱性（エスケープされていないユーザー入力）
- パストラバーサル（サニタイズされていないファイルパス）
- CSRF 保護の欠如
- 認証バイパス

### コード品質

- 大きな関数（50行超） - 小さく分割する
- 大きなファイル（800行超） - モジュールを抽出する
- 深いネスト（4レベル超） - 早期リターンを使用する
- エラーハンドリングの欠如 - 明示的にハンドリングする
- ミューテーションパターン - イミュータブル操作を優先する
- テストの欠如 - テストカバレッジを追加する

### パフォーマンス

- N+1 クエリ - JOIN またはバッチ処理を使用する
- ページネーションの欠如 - クエリに LIMIT を追加する
- 制限のないクエリ - 制約を追加する
- キャッシュの欠如 - 高コストな操作をキャッシュする

## 承認基準

- **承認**: CRITICAL または HIGH の問題がない
- **警告**: HIGH の問題のみ（注意してマージ）
- **ブロック**: CRITICAL の問題が見つかった
